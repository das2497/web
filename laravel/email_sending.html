Email Sending in Laravel

Step-by-step guide to sending emails using Laravel's built-in Mail functionality

Sending emails in Laravel is straightforward using the built-in Mail functionality. Here's how you can do it step by step:

1. Configure Mail Settings

Laravel uses Mailtrap, SMTP, Sendmail, Amazon SES, and other services for sending emails. Set up your .env file with your mail configuration:

Mailtrap Configuration

MAIL_MAILER=smtp MAIL_HOST=smtp.mailtrap.io MAIL_PORT=2525 MAIL_USERNAME=your_mailtrap_username MAIL_PASSWORD=your_mailtrap_password MAIL_ENCRYPTION=tls MAIL_FROM_ADDRESS=your@email.com MAIL_FROM_NAME="Your App Name"

Gmail SMTP Configuration

MAIL_MAILER=smtp MAIL_HOST=smtp.gmail.com MAIL_PORT=587 MAIL_USERNAME=your_gmail@gmail.com MAIL_PASSWORD=your_gmail_app_password MAIL_ENCRYPTION=tls MAIL_FROM_ADDRESS=your_gmail@gmail.com MAIL_FROM_NAME="Your Name"

Important

If using Gmail, enable Less Secure Apps or generate an App Password in Google security settings.

2. Create Mailable Class

Run the following command to generate a Mailable class:

php artisan make:mail MyEmail

This creates app/Mail/MyEmail.php. Modify it like this:

namespace App\Mail; use Illuminate\Bus\Queueable; use Illuminate\Contracts\Queue\ShouldQueue; use Illuminate\Mail\Mailable; use Illuminate\Queue\SerializesModels; class MyEmail extends Mailable { use Queueable, SerializesModels; public $details; // Data to pass to the email view public function __construct($details) { $this->details = $details; } public function build() { return $this->from('your@email.com') ->subject('Test Email') ->view('emails.myemail') ->with('details', $this->details); } }

3. Create Email Blade View

Inside resources/views/emails/myemail.blade.php:

<!DOCTYPE html> <html> <head> <title>Email from Laravel</title> </head> <body> <h1>{{ $details['title'] }}</h1> <p>{{ $details['message'] }}</p> </body> </html>

4. Send Email

Now, send an email from your controller:

use App\Mail\MyEmail; use Illuminate\Support\Facades\Mail; public function sendEmail() { $details = [ 'title' => 'Welcome to Laravel Mail', 'message' => 'This is a test email from Laravel.' ]; Mail::to('recipient@email.com')->send(new MyEmail($details)); return "Email sent successfully!"; }

5. Testing Email

Run your Laravel project and hit the route that triggers sendEmail(). If everything is set up correctly, the recipient should receive the email.

Pro Tip

Would you like to queue emails for better performance? 🚀 Use Mail::to('recipient@email.com')->queue(new MyEmail($details)); instead of send().
