Multi-Auth Implementation in Laravel

Step-by-step guide to implementing multiple authentication systems

Create Project

composer create-project --prefer-dist laravel/Laravel [name]

Install Laravel Breeze

composer require laravel/breeze --dev
php artisan breeze:install blade
php artisan migrate
npm install
npm run dev

Implementation Steps

1

User Migration - Add the "role" column

Add a role column to your users table migration:

$table->enum('role', ['admin','user'])->default('user');
2

Edit RegisteredUserController

Modify the create method to include the role field:

protected function create(array $data) {
return User::create([ 'name' => $data['name'],
'email' => $data['email'],
'password' => Hash::make($data['password']),
'role' => $data['role'], ]);
}
3

Edit AuthenticatedSessionController

Modify the login redirect logic based on user role like this.

if (Auth::user()->role === 'admin') {
return redirect('/admin/dashboard');
} elseif (Auth::user()->role == 'user') {
return redirect('/user/dashboard');
} else {
return redirect('/');
}
4

Create Login Middleware

Create middleware to check if user is logged in:

php artisan make:middleware Login

Create other user middlewares (e.g., Admin, User):

php artisan make:middleware Admin
php artisan make:middleware User
5

Bootstrap => app.php

Register your middleware in app/Http/Kernel.php:

protected $middlewareAliases = [
'login' => \App\Http\Middleware\Login::class,
'admin' => \App\Http\Middleware\Admin::class,
'user' => \App\Http\Middleware\User::class,
];
